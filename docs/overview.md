# Обзор проекта

## Что такое lazy-sandbox

**lazy-sandbox** — веб-приложение для интерактивного сравнения шаблонизаторов JavaScript.
Пользователь вводит шаблон и JSON-данные, выбирает движок (Handlebars, Mustache или EJS),
нажимает «Рендер» — и видит результат вместе со временем выполнения.
Каждое измерение автоматически сохраняется на сервере, формируя историю бенчмарков.

## Цели проекта

- Дать разработчикам простой инструмент для сравнения синтаксиса и скорости популярных шаблонизаторов.
- Предоставить REST API для накопления и анализа статистики рендеринга.
- Послужить учебным примером full-stack приложения (React + FastAPI + SQLite).

## Поддерживаемые шаблонизаторы

| Движок | Синтаксис переменной | Особенности |
|--------|----------------------|-------------|
| **Handlebars** | `{{name}}` | Блоки `{{#if}}`, `{{#each}}`, хелперы |
| **Mustache** | `{{name}}` | Логически-безвредный (logic-less), `{{#section}}` |
| **EJS** | `<%= name %>` | JavaScript-выражения, `<% %>` для управления |

## Ключевые характеристики

- **Рендеринг на стороне клиента** — все три движка работают в браузере; бэкенд не участвует в рендеринге.
- **Измерение времени** — используется `performance.now()` с точностью до 0.0001 мс.
- **Автосохранение** — после каждого успешного рендера результат отправляется на `POST /benchmarks`.
- **История** — таблица бенчмарков на странице обновляется после каждого сохранения.
- **Встроенная документация API** — Swagger UI доступен на `/docs`, ReDoc на `/redoc`.
- **Лёгкое развёртывание** — SQLite не требует отдельного процесса БД.
